# Systemd Service Template for Profile Gateways

Auto-generated by `hermes gateway install --profile NAME`. Template for reference:

## Service File Location

```
~/.config/systemd/user/hermes-gateway-NAME.service
```

## Template

```ini
[Unit]
Description=Hermes Agent Gateway - Messaging Platform Integration
After=network-online.target
Wants=network-online.target

[Service]
Type=simple
ExecStart=/home/hermes/.hermes/hermes-agent/venv/bin/python -m hermes_cli.main --profile NAME gateway run --replace
Environment="PATH=/home/hermes/.hermes/hermes-agent/venv/bin:/home/hermes/.hermes/hermes-agent/node_modules/.bin:/home/hermes/.hermes/node/bin:/home/hermes/.local/bin:/usr/local/sbin:/usr/local/bin:/usr/sbin:/usr/bin:/sbin:/bin"
Environment="VIRTUAL_ENV=/home/hermes/.hermes/hermes-agent/venv"
Environment="HERMES_HOME=/home/hermes/.hermes"
Restart=always
RestartSec=5

[Install]
WantedBy=default.target
```

Key: the `--profile NAME` flag in `ExecStart` tells Hermes which profile directory to use.

## Symlink (auto-created)

```
~/.config/systemd/user/default.target.wants/hermes-gateway-NAME.service
→ ~/.config/systemd/user/hermes-gateway-NAME.service
```

This enables auto-start on boot (requires `loginctl enable-linger $USER`).

## Multiple Profiles Example

```
hermes-gateway.service           → default profile
hermes-gateway-mira.service      → mira profile (accountability bot)
hermes-gateway-project.service   → project profile (team bot)
```

Each runs as an independent process with its own PID, memory, and gateway loop.
